# OpenClaw Newspaper - Markdown 项目管理使用说明

## 概述

OpenClaw Newspaper 现在支持使用 Markdown 格式创建和管理项目。系统会自动将 Markdown 内容转换为报纸样式的排版，保持与现有报纸样式一致。

## 快速开始

### 1. 启动服务器

```powershell
# 在项目根目录运行
powershell -ExecutionPolicy Bypass -File start_servers.ps1
```

这会同时启动：
- 主 HTTP 服务器（端口 8000）
- 管理 API 服务器（端口 8001）

### 2. 访问项目管理界面

- **主门户**：http://localhost:8000/site/index.html
- **项目管理**：http://localhost:8000/site/admin/index.html

您也可以通过主门户页面右上角的"项目管理"按钮直接跳转。

## Markdown 格式要求

项目信息必须按照以下 Markdown 格式编写：

```markdown
# 项目名称

## 摘要
项目摘要内容...

## 阶段
当前阶段描述...

## 阻塞与问题
阻塞问题描述...

## 下一步计划
下一步计划内容...

## 正文内容
这里是主要的报纸正文内容，可以使用：
- 列表
- **粗体**
- *斜体*
- 等等 Markdown 语法
```

### 必需的元素

1. **项目名称**（`# 项目名称`）
   - 作为项目的主标题
   - 自动生成项目 slug

2. **摘要**（`## 摘要`）
   - 项目简介
   - 显示在项目卡片和项目页面

3. **阶段**（`## 阶段`）
   - 当前项目阶段
   - 如：开发阶段、测试阶段等

4. **阻塞与问题**（`## 阻塞与问题`）
   - 当前遇到的问题或阻塞
   - 如果没有，可以写"暂无阻塞"

5. **下一步计划**（`## 下一步计划`）
   - 下一阶段的工作计划
   - 具体的行动项

6. **正文内容**（`## 正文内容`）
   - 主要的报纸正文
   - 可以使用丰富的 Markdown 语法

### 支持的 Markdown 语法

- **标题**：`# H1`, `## H2`, `### H3`
- **粗体**：`**文本**`
- **斜体**：`*文本*`
- **列表**：`- 列表项`
- **段落**：空行分隔

## 使用流程

### 创建新项目

1. 打开项目管理界面
2. 在左侧的 Markdown 编辑器中输入项目信息
3. 右侧会实时预览转换后的效果
4. 确认无误后，点击"创建项目"按钮
5. 系统会自动：
   - 创建项目目录
   - 生成项目元数据
   - 创建报纸样式的 HTML 文件
   - 更新门户索引

### 查看项目列表

1. 点击"项目列表"标签
2. 查看所有现有项目
3. 点击"查看"按钮可以打开项目页面

### 编辑现有项目

目前版本支持创建新项目。如需编辑现有项目，请：
1. 找到项目目录
2. 编辑项目的 JSON 元数据文件
3. 重新运行渲染脚本

```markdown

## 阶段

## 阻塞与问题
暂无阻塞

## 下一步计划
完成需求文档，开始原型设计

## 正文内容
AI 助手项目旨在为用户提供智能化的对话服务。

### 核心功能
- 自然语言理解
- 知识库检索
- 上下文记忆

### 技术栈
- Python
- 机器学习模型
```

### 示例 2：复杂项目

```markdown
# 企业级数据分析平台

## 摘要
构建一个企业级的数据分析平台，支持多数据源接入、实时分析和可视化展示。

## 阶段
开发阶段

## 阻塞与问题
- 数据源接入接口需要进一步优化
- 实时处理性能有待提升

## 下一步计划
- 优化数据接入性能
- 完善可视化组件
- 添加用户权限管理

## 正文内容
企业级数据分析平台是一个综合性的数据处理和分析系统。

### 主要模块

#### 数据接入模块
支持多种数据源：
- 关系型数据库
- NoSQL 数据库
- API 接口
- 文件上传

#### 分析引擎
提供强大的数据处理能力：
- **实时流处理**：支持实时数据分析
- **批量处理**：支持大规模数据批处理
- **机器学习**：集成常用的机器学习算法

#### 可视化展示
丰富的图表类型：
- 折线图
- 柱状图
- 饼图
- 散点图

### 技术架构

系统采用微服务架构，主要技术包括：
- 后端：Python + FastAPI

- 数据库：PostgreSQL + Redis
- 消息队列：RabbitMQ
```

## 界面说明

### 主门户页面

- **项目总控台**：显示所有项目的概览
- **项目管理按钮**：右上角红色按钮，点击进入项目管理界面
- **项目卡片**：显示项目的基本信息

### 项目管理界面

- **创建项目标签**：用于创建新项目
- **项目列表标签**：查看所有现有项目
- **Markdown 编辑器**：左侧输入区域，支持实时预览
- **预览区域**：右侧显示转换后的效果
- **操作按钮**：
  - 预览效果：手动刷新预览
  - 清空：清空编辑器内容
  - 创建项目：提交创建请求

## 技术细节

### Markdown 转换

系统使用自定义的 Markdown 解析器，支持：
- 标题转换（H1-H3）
- 文本格式化（粗体、斜体）
- 列表处理
- 段落分隔

### 样式保持

转换后的 HTML 使用与现有报纸相同的 CSS 样式：
- 纸质背景色
- 衬线字体
- 专业的排版间距
- 响应式设计

### 项目数据结构

创建的项目包含以下信息：
- `slug`：项目唯一标识符（自动生成）
- `label`：项目名称
- `summary`：项目摘要
- `stage`：当前阶段
- `blockers`：阻塞与问题
- `next`：下一步计划
- `description`：详细描述
- `status`：项目状态

## 常见问题

### Q: 项目名称支持中文吗？
A: 支持。系统会自动将中文项目名称转换为合适的 slug。

### Q: 可以使用其他 Markdown 语法吗？
A: 目前支持基本的 Markdown 语法。如果需要更复杂的功能，可以手动编辑生成的 HTML 文件。

### Q: 创建项目后可以修改吗？
A: 目前版本主要支持创建。如需修改，可以直接编辑项目目录中的文件。

### Q: 如何删除项目？
A: 手动删除项目目录，然后重新运行渲染脚本更新门户。

### Q: 支持图片吗？
A: 当前版本不支持图片上传。如需添加图片，请手动编辑生成的 HTML 文件。

## 故障排除

### API 服务器无法启动
- 检查端口 8001 是否被占用
- 确保 Python 环境正常
- 查看终端错误信息

### 项目创建失败
- 检查 Markdown 格式是否正确
- 确认所有必需的元素都已填写
- 查看 API 服务器日志

### 预览不显示
- 刷新页面
- 检查浏览器控制台错误
- 确认 Markdown 内容不为空
